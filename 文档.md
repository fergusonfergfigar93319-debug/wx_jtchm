# 📄 健康生活 (Health Life) 后端测试环境搭建与启动指南

欢迎进行后端项目的测试工作！本项目后端基于 **Python 3.10+**（仓库当前依赖支持 **Python 3.13**，请与 `requirements.txt` 保持一致）+ **Django 4.2.x** 构建，数据库采用了 **MySQL + MongoDB** 双库架构。为了保证测试环境的一致性，请严格按照以下步骤从零开始配置。

## 🛠️ 第一阶段：环境准备 (Prerequisites)

在开始之前，请确保你的电脑已安装以下基础软件：

1. **Python**（推荐 **3.10～3.13**；若遇个别包编译失败，可改用 3.11）。
2. **Git**（用于拉取代码）。
3. **Docker Desktop**（强烈推荐！用于一键拉起 MySQL、MongoDB 和 Redis，免去繁琐的本地安装）。

---

## 🚀 第二阶段：项目初始化

### 1. 拉取代码

在终端（CMD 或 PowerShell）中执行：

```bash
git clone <这里填入项目的 Git 仓库地址>
cd health_life_backend_dj
```

### 2. 搭建 Python 虚拟环境 (Virtual Environment)

为了防止依赖冲突，必须在虚拟环境中运行项目。

```bash
# 1. 创建名为 venv 的虚拟环境
python -m venv venv

# 2. 激活虚拟环境 (Windows)
venv\Scripts\activate
# (如果是 Mac/Linux，请使用: source venv/bin/activate)

# 3. 升级 pip 并安装项目依赖
python -m pip install --upgrade pip
pip install -r requirements.txt
```

`requirements.txt` 中已包含 **PyMySQL** 与 **cryptography**（MySQL 8 默认 `caching_sha2_password` 时 PyMySQL 需要后者）。若你手动精简过依赖，遇到 `cryptography package is required for ... caching_sha2_password` 时执行：`pip install cryptography`。

*（注意：执行完毕后，命令行路径前面应该多出一个 `(venv)` 的标识）*

---

## ⚙️ 第三阶段：配置文件与数据库拉起

### 1. 环境变量（`.env`）

- 复制或对照项目中的示例，在项目根目录配置 **`.env`**。
- **`DB_PASSWORD`**：必须与下面 MySQL 实际使用的 root 密码一致。
  - 若使用 **Docker Compose** 启动 `db` 服务：`docker-compose.yml` 里 `MYSQL_ROOT_PASSWORD: ${DB_PASSWORD}` 会从 `.env` 读取，**三者同源**，改密码时只改 `.env` 并重建容器即可。
  - 若使用 **本机已安装的 MySQL**：请在 `.env` 中填写该实例上 `root`（或你创建的用户）的真实密码，或把数据库用户密码改成与 `.env` 一致。

### 2. 一键拉起数据库 (Docker 方式)

项目根目录下提供了 `docker-compose.yml`。请确保 Docker Desktop 已启动，然后在**项目根目录**执行：

**方式 A：完整栈（含 Django 容器 + Nginx）**

```bash
docker compose up -d
```

*（若你使用旧版 Docker，命令可能是 `docker-compose up -d`。）*

**方式 B：仅数据库（适合本机 `python manage.py runserver` 调试）**

不构建 Web 镜像，只启动 MySQL、Redis、MongoDB：

```bash
docker compose up -d db redis mongo
```

*（首次执行可能需要数分钟下载镜像。）*

---

## 📦 第四阶段：数据初始化

数据库启动且 **`.env` 中 MySQL 账号可连通**后，执行 Django 迁移与数据脚本。

请确保此时虚拟环境 `(venv)` 已激活，在项目根目录依次执行：

**说明：** 第 4、5 步依赖本机 **MongoDB 已监听 `127.0.0.1:27017`**（可用 Docker：`docker compose up -d mongo`）。未启动时这两条会跳过或提示 `[SKIP]`，不影响 MySQL 迁移与 `load_data` 的 MySQL 部分。

```bash
# 1. 生成并应用关系型数据库表结构 (MySQL)
python manage.py makemigrations
python manage.py migrate

# 2. 初始化后台管理系统的左侧菜单
python manage.py init_menus

# 3. 加载基础业务字典数据 (如标签、默认配置等)
python manage.py load_data

# 4. 导入初始食谱数据 (存入 MongoDB)
python scripts/import_full_recipes.py

# 5. 建立 MongoDB 的地理位置与检索索引 (重要，否则附近餐厅等功能报错)
python scripts/fix_mongo_index.py

# 6. 创建超级管理员账号 (根据提示输入账号、邮箱和密码)
python manage.py createsuperuser
```

## ⚡ 第五阶段：启动服务与真机联调

### 1. 启动本地后端服务

```bash
# 监听所有网卡，确保局域网可访问
python manage.py runserver 0.0.0.0:8000
```

启动成功后，浏览器访问 `http://127.0.0.1:8000/api/docs/` 可查看 Swagger API 接口文档。访问 `http://127.0.0.1:8000/admin/` 可登录管理后台。

### 2. 微信小程序真机联调穿透 (测试必读)

如果测试人员需要使用**手机微信**扫码进行“真机调试”，由于局域网隔离，手机无法直接访问电脑的 8000 端口。

**操作步骤**：

1. 保持上述的 Django `runserver` 黑窗口处于运行状态，**不要关闭**。
2. 在项目根目录找到并双击运行 `start_tunnel.bat` 脚本。
3. 按提示输入云服务器密码，建立 SSH 安全隧道。
4. 看到 `Starting SSH Tunnel... [Local:8000 -> Public:8081]` 且窗口保持静止，代表穿透成功。
5. 此时，前端小程序的请求会自动打到云服务器 `47.93.45.198:8081` 并转发回你的本地电脑，即可流畅进行真机测试。

*（如遇隧道闪退或前端提示超时，请重新运行 `start_tunnel.bat`）*

## ❓ 常见问题排查 (Troubleshooting)

* **Q: 执行 `pip install` 报错各种 C++ 编译错误？**  
  A: 尝试换用 Python 3.11；或按报错提示安装对应库的 **预编译 wheel**（升级 pip：`python -m pip install -U pip`）。

* **Q: 报错 `cryptography package is required for ... caching_sha2_password`？**  
  A: 执行 `pip install cryptography`，并确认 `requirements.txt` 未去掉该依赖。

* **Q: `Access denied for user 'root'@'localhost'` (1045) 或连不上 MySQL？**  
  A:  
  1. 若用 Docker：在 Docker Desktop 中确认 `health_life_mysql`（或 `db` 服务）已运行；`.env` 里 **`DB_PASSWORD` 必须与 Compose 使用的密码一致**（见上文第三节）。  
  2. 若用本机 MySQL：用客户端或命令行确认 `DB_USER` / `DB_PASSWORD` / `DB_HOST` / `DB_PORT` 能登录；与 `.env` 不一致则修改 `.env` 或修改数据库用户密码。  
  3. 若曾换过 `MYSQL_ROOT_PASSWORD` 且容器使用旧数据卷，可能需要调整密码或清理对应 volume 后重建（**会丢库，慎用**）。

* **Q: 小程序提示“登录失败，未返回凭证”或一直转圈？**  
  A: 请确保内网穿透脚本 `start_tunnel.bat` 已成功运行，且在微信开发者工具中已点击“清除全部缓存”后重新编译。

* **Q: 发现 Lottie 动态创建 canvas 组件报错？**  
  A: 这是微信小程序的已知警告，不影响后端业务逻辑运行，直接忽略即可。

## ✅ 调试自检顺序（建议）

1. `(venv)` 已激活，`pip install -r requirements.txt` 无报错。  
2. Docker 仅数据库：`docker compose up -d db redis mongo`（或你的等价环境）。  
3. `.env` 中 MySQL 配置与真实实例一致。  
4. `python manage.py migrate` 成功。  
5. 依次执行第四阶段 `init_menus` → `load_data` → 脚本 → `createsuperuser`。  
6. `runserver` 后访问 `/api/docs/`、`/admin/` 正常。
