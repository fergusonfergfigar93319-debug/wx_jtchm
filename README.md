# 今天吃什么 / 健康生活（Health Life）

中国高校计算机设计大赛相关项目。仓库内包含**微信小程序前端**与**Django 后端**（见各子目录与 `文档.md` 中的环境说明）。

---

## 一、产品与技术总览

| 层级 | 要点 |
|------|------|
| 客户端 | 微信小程序；AI 对话（`packageAI/ai-chat`）、营养摄入记录与业务闭环 |
| 服务端 | Python 3.10+、Django 4.2.x；MySQL + MongoDB（+ Redis）；详见 `文档.md` |
| AI 链路 | 后端聚合大模型能力；前端通过**结构化锚点**驱动可视化组件 |

---

## 二、前端体验与工程亮点（参赛叙事可参考）

以下表述可与答辩材料中的「人机交互 / 前端工程」章节对应。

1. **流式打字机与视觉状态机**  
   对接 SSE/流式片段时，以 `isStreaming` 等状态驱动正文逐字呈现，并配合光标闪烁；输入区**操作栏**在输出过程中降低不透明度（如约 `0.34`），结束后恢复为 `1`，在**不增加额外点击层级**的前提下完成视觉降噪。

2. **动效与触控反馈**  
   关键过渡采用 `cubic-bezier` 等缓动曲线；可点击卡片使用 `transform: scale(0.96)` 等**按压态**，提升移动端「可点可预期」的质感。

3. **营养仪表盘：从非结构化到可渲染**  
   对助手回复做正则解析，捕获形如 `[Kcal: xxx] [碳水: xxg] [蛋白质: xxg] [脂肪: xxg]` 的**英文方括号 + 冒号**锚点，并渲染为品牌色高亮的营养仪表盘卡片；解析失败时走**降级展示**（仅文本，不阻断阅读）。

4. **一键记录闭环**  
   用户点击仪表盘卡片触发 `applyNutrition`，携带已解析营养字段跳转摄入编辑页（如 `intake-edit`），实现「聊完即记」的数据闭环，减少二次录入摩擦。

5. **兼容与降级**  
   通过 `liteMode`（如鸿蒙/低性能设备）等开关，对重动画能力做裁剪，保证可用性优先。

## 三、AI 鲁棒性：后端 System Prompt（提示词约束工程）

大模型输出若偏离约定格式，前端仪表盘将无法稳定渲染。建议在 **`ai_service.py`（或等价调用层）** 将下列内容作为**高权重系统消息**，对输出做强约束（答辩可表述为 **Prompt Constraints Engineering**：Few-Shot + 类 Markdown 数据锚点，降低格式幻觉）。

```text
# Role
你是一名拥有临床营养学背景的「首席 AI 智慧饮食管家」。你专业、温暖、富有共情力。你的目标是为用户提供精准、可执行的饮食建议。

# Output Constraints (绝对指令)
为了配合前端可视化引擎的渲染，你的回复必须严格遵循以下规则，不可有任何偏差：
1. 语气要求：像朋友一样自然对话，语言精炼，适合在手机屏幕上快速阅读。避免使用晦涩的学术名词。
2. 数据锚点（Critical Requirement）：如果你的建议中包含具体的营养成分估算，必须在段落的末尾或相关句子的末尾，严格使用英文方括号和冒号的格式输出指标。
   - 格式规范：[Kcal: {数值}] [碳水: {数值}g] [蛋白质: {数值}g] [脂肪: {数值}g]
   - 错误示范：热量是500kcal，[卡路里: 500]
   - 正确示范：这份减脂晚餐非常棒。[Kcal: 450] [碳水: 40g] [蛋白质: 35g] [脂肪: 15g]
3. 时段与节气感知：如果用户提示了当前时间或节气，请在开头的一两句话中自然地带入关怀（例如：「晚上好，今天立夏…」）。
4. 行动闭环：在给出建议后，使用一两句话鼓励用户将该数据记录到系统中。

# Examples
User: 帮我看看这份番茄炒蛋盖饭热量高吗？
Assistant: 番茄炒蛋盖饭是一道经典的家常菜，但因为炒蛋通常需要较多油脂，再加上米饭，整体碳水和热量会偏高哦。不过别担心，偶尔吃一次完全没问题！我为你估算了一下这份餐食的数据：[Kcal: 650] [碳水: 80g] [蛋白质: 18g] [脂肪: 25g]。
如果你正在减脂期，下次可以尝试少油版本，或者把一半米饭换成粗粮。快把这餐记录下来吧，让我帮你追踪今天的总摄入！
```

**答辩话术示例（可自行压缩）：**  
在 AI 驱动模块中，除前端正则捕获外，后端通过**强约束提示词工程**，结合少样本示例与固定锚点语法，将自然语言稳定映射为前端可组件化渲染的结构化片段，形成端到端工程闭环。

## 四、演示视频分镜建议（约 1 分钟）

| 时段 | 画面与交互 | 字幕/旁白要点 |
|------|------------|----------------|
| 0–15s | 进入 `ai-chat`，展示毛玻璃与层次化 UI；欢迎语含时段/节气 | 系统初始化，AI 结合时段主动问候 |
| 15–40s | 点击底部快捷问题；打字机与光标；操作栏半透明；结束后变亮并弹出营养仪表盘 | 正则实时捕获锚点，无缝渲染仪表盘 |
| 40–60s | 点击仪表盘（按压缩放）；跳转 `intake-edit` 已预填；提交记录 | 一键记录闭环，形成数据飞轮 |
